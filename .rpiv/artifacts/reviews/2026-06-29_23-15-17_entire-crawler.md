## ≡ااة Important

### I1 ≡ااة Launch setup failure can strand a crawl run in `running`

**Where**
`apps/crawler/src/crawler.ts:230`

**Code**
```ts
  const { crawlRun } = startCrawlRun(config.db, {
```

**Why**
`startCrawlRun` creates a non-terminal row before `launchContext: await createCloakLaunchContext(config.headless)` at `apps/crawler/src/crawler.ts:274`. That await is before the guarded `try` at `apps/crawler/src/crawler.ts:302` and before `finishCrawlRun` at `apps/crawler/src/crawler.ts:317`, so launch-context failures bypass finalization.

**Fix**
Wrap launch-context construction and crawler execution in one lifecycle `try/finally` that always attempts to finalize the crawl run.

### I2 ≡ااة Finalization errors can mask crawler failures and skip cleanup

**Where**
`apps/crawler/src/crawler.ts:317`

**Code**
```ts

```

**Why**
`finishCrawlRun` runs before `await crawler.teardown().catch(() => undefined);` at `apps/crawler/src/crawler.ts:328` and before `throw runError;` at `apps/crawler/src/crawler.ts:331`. If finalization throws, the original crawler error is replaced and teardown is skipped.

**Fix**
Move teardown into a `finally`; preserve and rethrow the original crawler error while logging/folding finalization errors separately.

**Where**
`apps/crawler/src/crawler.ts:124`

**Code**

### I4 ≡ااة Site-level quality failures can be recorded as successful crawl runs

**Where**
`apps/crawler/src/crawler.ts:315`

**Code**
```ts
  const finalStatus = runError || counters.failed > 0 ? "failed" : "succeeded";
```

**Why**
`runComicSiteCrawler` finalizes based only on Crawlee failures. The site scripts then enforce semantic gates such as `if (summary.comicsStored < 1)` at `apps/crawler/src/scripts/rouman5.ts:90` and `apps/crawler/src/scripts/18comic.ts:90`; when those gates throw, production only marks `running` rows failed at `apps/crawler/src/scripts/production.ts:665`, leaving the already-terminal crawl run as `succeeded`.

**Fix**
Move zero-comic/zero-chapter quality gates into the shared runner finalization path or let scripts update the crawl run to `failed` before exiting non-zero.

### I5 ≡ااة Production summaries drop completed run records when a quality gate throws

**Where**
`apps/crawler/src/scripts/production.ts:696`

**Code**
```ts
  const records: RunRecord[] = [];
```

**Why**
`runSelectedSites` pushes a probe/full `RunRecord` locally, then can throw on quality findings at `apps/crawler/src/scripts/production.ts:703` or `:714`. The caller assigns `records = await runSelectedSites(...)` only after the promise resolves at `apps/crawler/src/scripts/production.ts:848`, so a thrown quality gate discards already-collected records from the summary JSON and webhook.

**Fix**
Keep the records array in the caller and pass it into `runSelectedSites`, or throw a structured error that carries partial records/findings.

### Q1 ≡ا¤╡ Caller-owned SQLite handles can bypass foreign-key enforcement

**Where**
`packages/db/src/client.ts:25`

**Code**

**Why**
